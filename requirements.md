# Swarm vs Hunter v3 — 要件定義書

> 「死ぬたびに化ける。化けたら群れる。群れたら喰う。」

## 1. 概要

2人対戦のMinecraftミニゲーム。**Swarm**（群れの王）と**Hunter**（狩人）に分かれて戦う。

### 30秒ルール

- **Swarm** はHP1。フィールドのmobに殴られるとそのmobに変身する（mobからのダメージでは死なない）。同種mobがSwarmに追従し、Swarmが攻撃した対象を群れで襲う。
- **Hunter** は鉄装備。Swarmを殺せば人間に戻せる。
- mobを倒すと卵を落とす。卵を割ると味方mobが出てくる。
- **Swarm勝利**: mob状態でHunterを倒す。**Hunter勝利**: Swarmを3回倒す or 3分生き残る。

---

## 2. 技術基盤

| 項目 | 内容 |
|------|------|
| プラットフォーム | Spigot / Paper Plugin |
| Minecraftバージョン | 1.21.x |
| 対戦人数 | 2人 (Swarm × 1, Hunter × 1) |

---

## 3. フィールド

- **サイズ**: 60×60〜80×80ブロック
- **構造**: 壁で囲まれた閉鎖空間
- **地形生成**: プラグイン内でのプロシージャル生成（壁・障害物・通路をランダム配置し、毎回異なるマップを生成）
  - 将来的にLLM等による凝った地形デザインにも対応可能な設計にする
- **環境ギミック**: なし（ブロック破壊・設置は不可）

---

## 4. ゲームフロー

```
コマンド実行 (/svh start)
  ↓
mob選択フェーズ（インベントリGUI）
  - Swarm: 2体選択
  - Hunter: 2体選択
  - 必ず1枠は非戦闘mob（各プレイヤーの選択に固定枠1つ）
  - 選択状況はお互いにリアルタイムで見える
  ↓
カウントダウン（フィールド生成・mob配置）
  ↓
ゲーム開始
  - 3分タイマー開始
  ↓
勝敗判定
  - Swarm勝利: Hunterを倒した
  - Hunter勝利: Swarmを3回倒した or 3分経過
  ↓
結果表示 → 終了
```

### UI
- **初期実装**: インベントリGUI（チェストUIにmobのスポーンエッグを並べてクリックで選択）
- **将来**: モーダルで凝ったデザインのGUI

---

## 5. mob選択

- 各プレイヤーが **2体ずつ**、合計 **4種** を選択する
- 各プレイヤーの2枠のうち **1枠は非戦闘mob固定**（豚、牛、羊、鶏など）
- 選択可能mob: バニラの全mobから選択可能（バランス調整は行わない。強いmob・弱いmobの取捨選択はプレイヤーの駆け引きに委ねる）
- **Swarmの戦略**: 強いmob（ゾンビ、スケルトン等）を選び、変身先の選択肢を強化したい
- **Hunterの戦略**: 弱いmob（豚、亀等）を選び、Swarmの変身先を弱体化させたい

---

## 6. 中立mobシステム

- フィールドに配置される全てのmobは **中立** として振る舞う
- バニラで敵対するmob（ゾンビ、スケルトン等）もこのフィールド内では中立
- mobは攻撃されない限り、SwarmにもHunterにも何もしない
- **環境ダメージ無効**: フィールド内の全mobは環境ダメージ（雨、水、日光、ディメンション不適合によるゾグリン化等）を受けない。Swarm変身中のプレイヤーも同様
  - 対象例: ブレイズの雨/水ダメージ、エンダーマンの雨/水ダメージ+テレポ暴走、ホグリンのゾグリン化、ゾンビ/スケルトンの日光炎上
- 選択された4種が **同じ確率** でフィールド全体にランダムスポーンする

---

## 7. 卵システム

### 基本仕組み
1. 中立mobを倒すと **卵を1個ドロップ**
2. 卵を使用すると **確定で味方mobが1体出現**（倒したmobと同種）
3. 味方mobは使用したプレイヤーについてくる

### 使用者別の用途
| プレイヤー | 用途 |
|-----------|------|
| Hunter | 護衛・偵察として味方mobを従える |
| Swarm | 囮・撹乱に使える |

### 寝取りメカニクス
- Hunterの味方mobの近くにSwarmが来た場合、**Swarmがその味方mobと同種のmobに変身中であれば**、その味方mobはSwarm側に寝返る
- Hunterは味方mobを寝取られないよう、Swarmの変身先に注意する必要がある

---

## 8. Swarm — 群れの王

### 基本ステータス
- **常にHP1**（人間でもmob状態でも一撃で死ぬ）
- 人間状態では戦闘力なし。逃げるか、mobに殺されるかの二択

### 変身メカニクス
1. 中立mobを殴って敵対させる → mobに殴られる → **そのmobに変身**（Swarmはmobに殺されない。ダメージを受けず変身する）
2. 変身すると、半径内の同種mobが **Swarmに追従する**（飼い慣らした狼のようについてくる）
3. Swarmが何かを攻撃すると、追従中の同種mobも **同じターゲットを攻撃する**（狼がオーナーの攻撃対象を襲うのと同じ）
4. 変身中に別種のmobに殴られれば、**そのmobに変身先を切り替え**（前の追従mobは中立に戻る）
5. Swarmが死ぬのは **Hunterに倒されたとき** と **落下ダメージなどの環境ダメージ** のみ。mob攻撃では死なない

### 変身中の性能
- **見た目**: 変身先mobと同じ外見。ネームタグなし
- **攻撃力**: 変身先mobのバニラ攻撃力
- **移動速度**: 変身先mobのバニラ移動速度

### 変身中の共通ルール
- **クールタイムなし**: 全ての能力にクールタイムは設けない。連打・連射可能
- **自傷しない**: 自分の爆発・範囲攻撃でSwarm自身はダメージを受けない
- **味方を巻き込まない**: 範囲攻撃は追従中の味方mobを除外する
- **環境ダメージ耐性**: フィールド内の環境ダメージ無効ルール（§6参照）に加え、変身先の特性に応じた追加耐性を付与（スライムの落下無効、スパイダーの落下無効等）

### mob別能力一覧

方針: **そのmobがバニラでできることを、プレイヤーが変身中にそのままできるようにする。**
独自スキルを発明しない。バニラの再現に徹する。

#### 戦闘mob

| mob | 能力タイプ | 仕様 |
|-----|-----------|------|
| **ゾンビ** | パッシブ | 殴るだけ |
| **スケルトン** | アイテム付与 | 弓+矢（無限）をインベントリに付与。普通に弓で戦える |
| **クリーパー** | 右クリック | 右クリで爆発。Swarm自身はダメージ0。味方mob除外 |
| **スパイダー** | パッシブ | 壁登り付与（SPIDER_CLIMB相当）。落下ダメージ無効 |
| **洞窟グモ** | パッシブ | 壁登り付与。攻撃ヒット時にHunterに毒付与（5秒） |
| **エンダーマン** | アイテム付与 | エンダーパールをインベントリに付与（5個）。使うとTP |
| **ウィッチ** | アイテム付与 | スプラッシュポーション数種をインベントリに付与（毒、鈍化、ダメージ） |
| **ブレイズ** | パッシブ+右クリック | 接触した敵に炎上付与（3秒）。右クリでファイアボール射撃 |
| **スライム** | パッシブ | 跳躍力UP（Jump Boost II）。落下ダメージ無効。接触ダメージ（0.5秒間隔で近接2ブロック内の敵にダメージ） |
| **ゾンビピグリン** | パッシブ | 殴るだけ |
| **ピグリン** | アイテム付与 | クロスボウ+矢（無限）をインベントリに付与 |
| **ホグリン** | パッシブ | 攻撃ヒット時に高ノックバック付与 |
| **ヴィンディケーター** | アイテム付与 | 鉄の斧をインベントリに付与。そのまま殴る |
| **ピリジャー** | アイテム付与 | クロスボウ+矢（無限）をインベントリに付与 |
| **ラヴェジャー** | 右クリック | 右クリで咆哮（周囲6ブロックのHunterのみノックバック+ダメージ）。味方mob除外 |
| **ヴェックス** | パッシブ | 飛行付与（クリエイティブ飛行と同じ操作）+鉄剣付与。時間制限なし |
| **ウォーデン** | 右クリック | 右クリでソニックブーム（前方15ブロック貫通ビーム）。壁貫通 |

#### 非戦闘mob

非戦闘mobはSwarmを殺せない（Swarmはmob攻撃で死なず変身する）ため、変身先にならない。能力の定義は不要。

### 能力タイプの説明

- **パッシブ**: 変身した瞬間に自動付与。操作不要
- **アイテム付与**: 変身時にインベントリにアイテムが入る。プレイヤーが普通に使う
- **右クリック**: 右クリックで発動。クールタイムなし

### 紛れる戦略
- Swarmに追従する同種mobの群れの中にSwarm本体も混じる
- SwarmがHunterを攻撃すれば群れも一斉にHunterを襲う
- Hunterから見れば「こっちに来るゾンビ6体のうち、どれがSwarm？」
- 中立mobも同じ見た目でフィールドにいるため、さらに見分けにくい

### 死亡時
- Hunterに倒されると **人間に戻る**
- 敵対化していたmobは **中立に戻る**
- **その場でリスポーン**。再度mobを探して変身し直す

## 9. Hunter — 狩人

### 初期装備
| スロット | アイテム |
|---------|---------|
| 頭 | 鉄ヘルメット |
| 胴 | 鉄チェストプレート |
| 脚 | 鉄レギンス |
| 足 | 鉄ブーツ |
| メインハンド | 鉄剣 |
| オフハンド | 盾 |
| インベントリ | 弓、矢×16、パン×4 |

### Hunterにできること
- mobを倒して卵を集める → 味方mob作成（護衛・偵察）
- mob群を観察してSwarmの位置を推理する → 敵対化した瞬間に動きが変わるmob群を見つける
- 人間状態のSwarmを先に見つけて潰す → 変身される前が一番安全

## 10. 勝利条件

| 勝者 | 条件 |
|------|------|
| **Swarm** | Hunterを倒す |
| **Hunter** | Swarmを **3回** 倒す |
| **Hunter** | **3分間** 生き延びる（タイムアップ） |

---

## 11. バランスパラメータ（初期値）

全てconfig（設定ファイル）で調整可能にする。

| パラメータ | 初期値 | 説明 |
|-----------|--------|------|
| `field.size` | 70 | フィールドの一辺のブロック数 |
| `field.mob_count_per_type` | 8 | 各mob種のスポーン数 (4種×8体=32体) |
| `swarm.aggro_radius` | 20 | 変身時に同種mobが追従を開始する範囲（ブロック） |
| `swarm.max_deaths` | 3 | Hunterがこの回数Swarmを倒せば勝利 |
| `game.duration_seconds` | 180 | ゲーム制限時間（秒） |
| `hunter.arrow_count` | 16 | Hunterの初期矢数 |
| `hunter.food_count` | 4 | Hunterの初期食料数（パン） |
| `egg.ally_count` | 1 | 卵1個から出現する味方mobの数 |
| `egg.steal_radius` | 10 | 寝取りが発動する範囲（ブロック） |

## 12. デバッグモード（ソロテスト用）

開発・テスト時に1人でゲームを起動できるモード。2人揃えなくても各機能を検証できる。

### コマンド

```
/svh debug <swarm|hunter>
```

- **OP権限必須**
- 通常の `/svh start` とは別コマンド

### 動作仕様

1. コマンド実行者が指定した役割（Swarm or Hunter）になる
2. 相手プレイヤーは不在（null扱い）
3. mob選択GUIが開き、**自分の分の4体を選択**してスタート
4. カウントダウン → フィールド生成 → mob配置 → ゲーム開始（通常と同じ流れ）

### 通常モードとの違い

| 項目 | 通常モード | デバッグモード |
|------|-----------|--------------|
| 必要人数 | 2人 | 1人 |
| mob選択 | 各自2体ずつ | 1人で4体選択 |
| 勝敗判定 | 有効 | 無効（タイマー・キルカウント非表示） |
| 終了方法 | 勝敗確定時 | `/svh stop` で手動終了 |
| 相手プレイヤー | 必須 | 不在（関連処理をスキップ） |

### 用途別テストシナリオ

- **`/svh debug swarm`**: 変身メカニクス、mob追従、固有能力、寝取りの検証
- **`/svh debug hunter`**: 装備、mob戦闘、卵システム、フィールド移動の検証
